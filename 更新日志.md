# 更新日志

本文件记录 `gpu-services` GPU 推理服务的主要变更。

## [V1.2.0] - 2026-06-02

### 关键修复

- **修复 ONNX Runtime CUDA 依赖不匹配**:
  - 将运行镜像从 `nvidia/cuda:12.1.1-cudnn8-runtime-ubuntu22.04` 调整为 `nvidia/cuda:11.8.0-cudnn8-runtime-ubuntu22.04`。
  - 当前依赖固定为 `onnxruntime-gpu==1.18.0`，该版本 PyPI GPU 包需要 CUDA 11.x 运行库，避免运行时报 `libcublasLt.so.11` 缺失。
  - 保留 Python 3.12，不降级到 Python 3.10。

- **修复日志时区不对齐**:
  - 在镜像和 Compose 环境变量中设置 `TZ=Asia/Shanghai`。
  - 容器日志和应用日志统一输出东八区时间。

- **优化镜像构建耗时**:
  - Dockerfile 中将 Ubuntu apt 源切换到清华镜像。
  - 使用 `python3.12 -m ensurepip --upgrade` 初始化 pip，避免访问 `bootstrap.pypa.io`。
  - 新增 `models.yml` 复制到镜像内 `/workspace/models.yml`。

### 业务推理接口

- **新增 `/infer/persons`**:
  - 支持 `multipart/form-data` 上传单张或多张图片。
  - 服务内完成图片解码、letterbox、YOLO 推理、person 类过滤和 NMS。
  - 返回轻量化人体检测结果：`box`、`score`、`class_id`、`class_name`。
  - 支持 batch 推理；当模型不支持动态 batch 时自动降级为逐帧推理。

- **新增 `/infer/person-embeddings`**:
  - 接收人体 crop 图片，服务端完成 ReID 预处理和 OSNet 推理。
  - 支持 ImageNet normalize 和 L2 embedding normalize。
  - 可通过 `include_vectors` 控制是否返回完整向量。

- **新增 `/infer/person-tracks`**:
  - 面向图片/多帧图片的检测 + ReID 组合接口。
  - 先检测每帧人体，再裁剪人体生成 ReID embedding。
  - 返回每帧 `persons` 列表，并为每个有效人体附加 `embedding_index`、`embedding_dim`，可选附加完整 `embedding`。
  - 不伪造跨帧 `track_id`，后续调用方可基于 embedding 与时间序列自行关联，或接入 ByteTrack/BoT-SORT。

- **保留 `/predict`**:
  - 继续作为通用 ONNX raw tensor 调试接口。
  - 响应和日志新增 `input_shape`、`output_shapes`，便于排查调用方是否只传入 batch size 1 或只解析第一个输出。

### 图片、离线视频和视频流入口

- **图片推理**:
  - `/infer/persons` 用于人体检测。
  - `/infer/person-embeddings` 用于人体特征提取。
  - `/infer/person-tracks` 用于检测 + ReID 组合分析。

- **离线视频推理**:
  - 新增 `/infer/video/person-tracks`。
  - 支持上传视频文件，服务端使用 OpenCV 抽帧。
  - 支持 `frame_interval` 和 `max_frames` 参数。
  - 返回 `source_frame_index` 和 `source_seconds`，便于调用方落库和回放定位。

- **视频流推理**:
  - 新增 `/infer/stream/person-tracks`。
  - 支持 `rtsp://`、`rtmp://`、`http://`、`https://` 视频源。
  - 默认关闭服务端主动拉流能力，需要设置 `ALLOW_STREAM_URLS=true` 后启用。
  - 支持 `frame_interval`、`max_frames`、`read_timeout_seconds`，避免单次请求长期占用 worker。

### 模型配置

- **新增 `models.yml`**:
  - 显式声明模型类型、输入尺寸和后处理参数。
  - 默认配置包含：
    - `cross_camera_tracking/yolov8n.onnx`: YOLO person 检测。
    - `cross_camera_tracking/osnet_ibn_x1_0.onnx`: OSNet ReID。

- **新增 `MODEL_CONFIG_PATH`**:
  - 容器内默认 `/workspace/models.yml`。
  - 本地直接运行默认读取当前目录 `models.yml`。
  - 可通过 `.env` 覆盖。

- **新增 `/model-configs`**:
  - 返回当前加载的模型配置和配置文件路径。

### 可观测性与调试

- **新增 `/ready/deep`**:
  - 可检查配置模型是否存在。
  - 可选 `load_models=true` 触发模型加载检查。
  - 可选 `dummy_inference=true` 执行 dummy inference，确认 CUDA session 可实际推理。

- **新增 `/debug/model-output`**:
  - 上传一张图，返回模型输入 shape、输出 shape、dtype、min/max 和少量 sample 值。
  - 用于排查 ONNX 导出格式，不返回完整大 tensor。

- **扩展 Prometheus 指标**:
  - 新增 persons、embeddings、tracks 请求数和错误数。
  - 新增检测人数、处理帧数。
  - 新增图片/视频解码、预处理、后处理耗时累计。

- **扩展结构化日志**:
  - 业务接口记录 `decode_seconds`、`preprocess_seconds`、`queue_seconds`、`inference_seconds`、`postprocess_seconds`、`total_seconds`。
  - 记录 `frame_count`、`person_count`、`embedding_count`、`inference_mode`。

### 调用方适配说明

- 调用方推荐优先对接 `/infer/person-tracks`，而不是旧的 `/predict`。
- 对于图片、离线视频、视频流三类业务，建议分别使用：
  - 图片/多帧图片：`/infer/person-tracks`
  - 离线视频：`/infer/video/person-tracks`
  - 视频流：`/infer/stream/person-tracks`
- `cross_camera_tracking` 这类调用方需要从旧的 JSON/base64 调用方式改为 `multipart/form-data` 文件上传方式。
- `GpuInferenceClient` 应适配新响应结构：
  - `frames[].persons[].box`
  - `frames[].persons[].score`
  - `frames[].persons[].class_id`
  - `frames[].persons[].embedding`
- 旧 `/predict` JSON tensor 链路保留为兼容和调试模式，不建议作为视频多人解析主路径。

### 已知边界

- 当前 `/infer/person-tracks` 返回检测结果和 ReID embedding，不生成最终跨帧 `track_id`。
- TensorRT、完整 ByteTrack/BoT-SORT 轨迹分配、长时间持续流 worker 调度暂未强行引入，后续可在当前接口和配置基础上继续扩展。
- 视频流拉取涉及内网安全，默认由 `ALLOW_STREAM_URLS=false` 禁用。
